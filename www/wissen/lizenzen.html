Softwarelizenzen

Zu jeder Computer-Software gibt es in der Regel einen Besitzer, der die Rechte daran hält. Der Besitzer ist dabei nicht zwangsläufig die Person, auf deren Computer die Software installiert ist. Stattdessen handelt es sich dabei häufig um den Urheber der Software. Das ist die Person, die den Quellcode programmiert hat.

Wenn der oder die Programmierer der Software für das Erstellen der Software bezahlt wurden, gehören die Rechte an dem Programm in der Regel dem Arbeitgeber. Das kann zum Beispiel die Firma sein, bei der die Programmierer angestellt sind. Natürlich kann der Besitzer einer Software diese auch an jemand anderen weiterverkaufen.

Die Programme oder Apps, die auf deinem Computer oder Smartphone installiert sind, gehören also nicht wirklich dir. Du hast lediglich die Nutzungsrechte dafür. Wenn du beispielsweise einen Computer verwendest, auf dem Windows installiert ist, dann ist Microsoft der Besitzer dieser Software. Natürlich gehören die Inhalte, die du auf einem solchen Computer selbst erstellst (also z. B. ein Programm, das du selber programmierst), trotzdem dir und nicht Microsoft.

Der Besitzer einer Software kann entscheiden, wofür diese verwendet werden darf. Er kann bestimmen, wer dazu berechtigt ist, sie zu benutzen, und ob man für die Nutzung etwas bezahlen muss. Außerdem kann er bestimmen, ob es erlaubt ist, die Software zu verändern oder sie weiterzuverbreiten. Er kann sogar verbieten, dass man die Software dekompiliert, also den Quellcode rekonstruiert, aus dem sie entstanden ist.

Um genau festzulegen, was erlaubt und was verboten ist, kann der Besitzer einer Software diese unter eine Lizenz stellen. Das ist ein Text, der genau auflistet, wozu man eine Software verwenden darf.

Es gibt verschiedene Arten von Softwarelizenzen. Man unterscheidet dabei zwischen zwei Kategorien: proprietären Lizenzen und freien Lizenzen.

Wenn eine Software unter einer proprietären Lizenz veröffentlicht wird, kann man sie in der Regel nicht einfach weiterverbreiten oder verändern. In vielen Fällen ist solche Software kostenpflichtig und oft ist es nicht einmal erlaubt, sich den Quellcode anzusehen, um zu verstehen, wie die Software genau aufgebaut ist.

Wenn die Software hingegen unter einer freien Lizenz steht, kann jeder die Software kostenlos nutzen und den dazugehörigen Quellcode lesen. Es ist dann auch erlaubt, den Code zu verändern und die Software an andere Menschen weiterzuverteilen. Wenn ein Programm unter einer freien Lizenz steht, ist es damit so etwas wie ein Allgemeingut, das der gesamten Menschheit zur Verfügung steht. Bekannte freie Lizenzen sind die GNU General Public Lizenz und die MIT-Lizenz.

Weil freie Software viele Vorteile gegenüber proprietärer Software aufweist, sollte man grundsätzlich versuchen, diese zu verwenden, wann immer es möglich ist. Nur wenn es aus anderen Gründen zwingend notwendig ist, sollte man auf proprietäre Closed-Source-Software zurückgreifen.
